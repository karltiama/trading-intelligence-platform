## Why e2e tests here

For this backend, e2e tests verify that a real HTTP request can flow through Nest routing, controller logic, and Prisma-backed services with the app booted as a whole system.

At Phase 0, these tests are intentionally small:

- `GET /` proves the app boots and basic routing works.
- `GET /health` proves your health contract stays stable.
- `GET /symbols` proves your API can serve DB-backed symbol rows.

This is not full coverage yet. It is a safety net for the core paths you rely on while building fast.

## How to run

From `apps/api`:

```bash
npm run test:e2e
```

If this passes, your route wiring and backend baseline behavior are still intact after changes.
