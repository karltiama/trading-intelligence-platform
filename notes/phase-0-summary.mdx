## Project purpose

Build a backend-first trading intelligence platform that supports market dashboards, automation, and paper-trading workflows with clean architecture and maintainable code.

## Stack

- Frontend: Next.js (`apps/web`)
- Backend: NestJS (`apps/api`)
- Database: Neon PostgreSQL
- ORM: Prisma
- Testing: Jest + Supertest e2e

## Architecture overview (current)

- Monorepo with app separation:
  - `apps/web` for UI and API consumption
  - `apps/api` for backend logic and data access
- Backend uses feature modules and Prisma integration via a global `PrismaModule`.
- Local API defaults to port `3001`; web defaults to `3000`.

## Current backend module responsibilities

- `health`: operational liveness endpoint (`GET /health`)
- `market-data`: current placeholder status endpoint (`GET /market-data/status`)
- `indicators`: service-only placeholder for indicator logic
- `signals`: placeholder signal listing endpoint (`GET /signals`)
- `scheduler`: service-only placeholder for future jobs
- app-level endpoint: `GET /symbols` reading from Neon via Prisma

## Current database models

- `Symbol`
  - instrument metadata (`ticker`, `name`)
- `Candle`
  - OHLCV by symbol/timeframe/timestamp
- `Signal`
  - strategy signal metadata tied to symbol
- `StrategyRun`
  - lifecycle tracking for strategy executions

## Testing foundation

- Unit tests for app baseline
- e2e tests covering:
  - `GET /`
  - `GET /health`
  - `GET /symbols`
- Prisma + Neon validated through migrations and data read/write checks.

## What Phase 0 accomplished

- Established monorepo app boundaries.
- Bootstrapped Nest modules with clear responsibilities.
- Integrated Prisma with Neon and applied initial migration.
- Seeded and read sample symbols (`AAPL`, `MSFT`, `SPY`).
- Added API route backed by real DB data (`GET /symbols`).
- Added practical developer workflow scripts and docs.

## Low-effort final cleanup decisions

- Keep canonical naming as `market-data` (kebab-case) across folders/routes/docs.
- Avoid refactors now; prioritize stable handoff and forward progress.
- Keep schema frozen unless directly required by Phase 1 scope.
