## Context

We initialized the frontend for the Trading Intelligence Platform using Next.js App Router in a monorepo path at `apps/web`. The goal was to use npm only and keep the scaffold aligned with our Cursor rules and MVP focus.

## Goal

Create a production-ready frontend baseline with:
- Next.js + TypeScript
- App Router
- Tailwind CSS
- ESLint
- npm scripts that run locally (`dev`, `build`, `lint`)

## Files Changed

- `apps/web/**` (generated by `create-next-app`)
- `apps/web/package.json`
- `apps/web/tsconfig.json`
- `apps/web/next.config.ts`
- `apps/web/src/**`
- `apps/web/public/**`

## Implementation Notes (key logic explained)

- Scaffold command used:
  - `npx create-next-app@latest web --ts --tailwind --eslint --app --src-dir --import-alias "@/*" --use-npm --yes`
- Created inside `apps/` so final app path is `apps/web`.
- Verified generated scripts in `package.json`:
  - `npm run dev`
  - `npm run lint`
  - `npm run build`
- Build verification passed, confirming TypeScript and Next compile pipeline are healthy from day one.

## Key Decisions (and alternatives considered)

- **Decision:** Use npm for bootstrap and install.
  - **Why:** Your explicit preference and simplest local setup.
  - **Alternative:** `pnpm` for faster monorepo installs (rejected for now because pnpm is not installed).
- **Decision:** Keep default Next App Router scaffold.
  - **Why:** Minimal, stable baseline with low setup risk.
  - **Alternative:** Custom folder conventions from start (rejected to avoid early complexity).

## Assumptions

- Monorepo structure remains `apps/web` + `apps/api`.
- Frontend will call backend over HTTP (no direct DB access).
- Tailwind is acceptable as the default styling system for MVP dashboards.

## Failure Modes

- Node version mismatch can break Next 16 install/build.
- Future root workspace config (`package.json` workspaces/turbo) may require script path updates.
- If backend API env vars are missing, future data-fetching pages will fail at runtime (not scaffold-time).

## How to Test (commands + steps)

```bash
cd apps/web
npm run dev
# open http://localhost:3000

npm run lint
npm run build
```

Manual checks:
1. Homepage loads at `http://localhost:3000`.
2. No lint errors.
3. Production build completes successfully.
