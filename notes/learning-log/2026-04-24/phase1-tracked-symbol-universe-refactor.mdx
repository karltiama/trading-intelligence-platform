## Context

Phase 1 already had Alpaca daily sync + `DailyPrice` persistence. The next step is controlling the symbol universe so we do not attempt to sync the entire market.

## Goal

Refactor backend sync to:
- track symbols in DB with an `isActive` flag,
- sync only active symbols,
- seed default symbols safely,
- expose simple symbol management endpoints.

## Implementation Notes (key logic explained)

- Added `Symbol.isActive` (default `true`) so symbol inclusion is data-driven.
- Added repository methods:
  - `seedDefaultSymbols()`,
  - `findActiveSymbols()`,
  - `listTrackedSymbols()`,
  - `createTrackedSymbol()`,
  - `toggleSymbolActive()`.
- Updated `POST /market-data/sync` behavior to:
  1) seed default symbols if missing,
  2) query active symbols,
  3) sync only those symbols,
  4) return aggregate summary: `{ message, symbolsProcessed, rowsUpserted }`.
- Added optional symbols endpoints:
  - `GET /symbols`,
  - `POST /symbols`,
  - `POST /symbols/bootstrap-defaults`,
  - `PATCH /symbols/:ticker/toggle`.

## Key Decisions (and alternatives considered)

- **Decision:** keep symbol management in a separate `symbols` module while reusing `MarketDataRepository`.
  - **Why:** keeps routes clear and avoids duplicating Prisma access.
  - **Alternative:** place symbol routes in `market-data.controller.ts` (simpler file count but mixes concerns).
- **Decision:** seed defaults inside `/market-data/sync` and via explicit `/symbols/bootstrap-defaults`.
  - **Why:** safe bootstrap and idempotent operation.
  - **Alternative:** only CLI/manual seed step (more setup friction).

## Assumptions

- `Symbol.assetType` is still optional and can be filled later.
- Existing `DailyPrice` pipeline remains source of truth for dashboard summary.
- Sync should currently stop on first hard failure (future enhancement: per-symbol failure reporting for active universe).

## Failure Modes

- If migration is not applied, `isActive` queries fail at runtime.
- Invalid ticker input (empty string) is rejected with `BadRequestException`.
- Alpaca/API errors during active symbol sync bubble up from current service flow.

## How to Test (commands + steps)

1. Apply migration:
   - `cd apps/api`
   - `npx prisma migrate deploy` (or `npx prisma migrate dev` in local dev)
2. Build + lint + test:
   - `npm run lint:api`
   - `npm run build:api`
   - `npm run test:api`
3. Start API:
   - `npm run dev:api`
4. Seed defaults:
   - `POST /symbols/bootstrap-defaults`
5. Verify tracked symbols:
   - `GET /symbols`
6. Toggle one symbol:
   - `PATCH /symbols/NVDA/toggle`
7. Run sync:
   - `POST /market-data/sync`
   - Confirm response has `message`, `symbolsProcessed`, and `rowsUpserted`.
