## Context

Phase 1 started with a requirement to prove clean Alpaca market-data integration without DB writes, while keeping Nest module boundaries production-minded.

## Goal

Create a new `src/modules/market-data` module with:
- reusable Alpaca client
- service methods for test + bars
- controller routes for `/market-data/test` and `/market-data/:symbol/bars`
- clean JSON output and proper Nest error handling.

## Implementation Notes (key logic explained)

- `AlpacaClient` centralizes API calls and headers from env variables.
- `MarketDataService` owns use-cases:
  - `testConnection()` uses `AAPL` latest quote
  - `getBars(symbol, limit)` fetches 1D bars (default 30)
- `MarketDataController` exposes:
  - `GET /market-data/test`
  - `GET /market-data/:symbol/bars`
- Responses are normalized (not raw Alpaca payloads).
- Errors from Alpaca are translated to `BadGatewayException`; missing credentials throw `InternalServerErrorException`.
- Added Nest `Logger` logs for successful quote/bar fetches.

## Key Decisions (and alternatives considered)

- **Decision:** use native `fetch` in client instead of adding axios/http module.
  - **Why:** fewer dependencies and enough for this scope.
  - **Alternative:** `HttpService` with RxJS pipeline (deferred to keep integration lean).

## Assumptions

- Alpaca credentials in `.env` are valid and market-data access is enabled.
- Using Alpaca IEX feed (`feed=iex`) is acceptable for MVP validation.

## Failure Modes

- Missing/invalid Alpaca credentials -> 500 with credentials/config error.
- Invalid symbol or upstream API errors -> 502 from Alpaca client wrapper.
