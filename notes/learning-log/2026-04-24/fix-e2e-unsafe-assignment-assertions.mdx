## Context

Strict lint rules flagged `expect.any(...)` matcher object usage as unsafe assignments in `app.e2e-spec.ts`.

## Goal

Keep `/symbols` e2e contract assertions while removing unsafe `any` assignments.

## Files Changed

- `apps/api/test/app.e2e-spec.ts`
- `notes/learning-log/2026-04-24/fix-e2e-unsafe-assignment-assertions.mdx`

## Implementation Notes (key logic explained)

- Replaced matcher-object assertion with explicit type checks:
  - `typeof first.id === 'string'`
  - same for `ticker`, `name`, `createdAt`, `updatedAt`

## Key Decisions (and alternatives considered)

- **Decision:** Use explicit field checks for strict lint compatibility.
  - **Alternative:** disable lint rule for the block (rejected).

## Assumptions

- `/symbols` returns ISO date strings in JSON payload.

## Failure Modes

- If response types change, this test will fail and surface contract drift.
