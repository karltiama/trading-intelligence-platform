## Context

Phase 0 wrap-up checks exposed strict lint errors in `app.e2e-spec.ts` because `response.body` was treated as `any`.

## Goal

Make the `/symbols` e2e test type-safe under current ESLint rules without changing endpoint behavior.

## Files Changed

- `apps/api/test/app.e2e-spec.ts`
- `notes/learning-log/2026-04-24/fix-e2e-symbols-lint-typing.mdx`

## Implementation Notes (key logic explained)

- Added a local `SymbolResponse` type for expected payload shape.
- Cast `response.body` to `SymbolResponse[]`.
- Replaced direct `response.body` usage with typed `symbols` variable.

## Key Decisions (and alternatives considered)

- **Decision:** use local response type in test file.
  - **Alternative:** disable lint rules inline (rejected to keep strict quality guardrails).

## Assumptions

- `/symbols` returns the selected fields defined in current service query.

## Failure Modes

- If `/symbols` response shape changes, this test will surface the mismatch.

npm run test:e2e
