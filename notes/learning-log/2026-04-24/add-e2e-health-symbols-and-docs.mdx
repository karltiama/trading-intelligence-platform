## Context

We added a DB-backed `GET /symbols` endpoint and already had `/health`. We needed end-to-end tests to guard these core Phase 0 backend paths.

## Goal

Extend e2e coverage for `GET /health` and `GET /symbols`, and add short docs introducing e2e purpose for this repo.

## Files Changed

- `apps/api/test/app.e2e-spec.ts`
- `notes/e2e-testing-intro.mdx`
- `notes/learning-log/2026-04-24/add-e2e-health-symbols-and-docs.mdx`

## Implementation Notes (key logic explained)

- Added `import 'dotenv/config';` in e2e spec so Prisma in test runtime uses `apps/api/.env` and hits Neon DB consistently.
- Added `/health` assertion for exact contract `{ status: 'ok' }`.
- Added `/symbols` assertion:
  - expects 200
  - expects array
  - validates object shape when array is non-empty
- Added short docs note describing why e2e exists and how to run it.

## Key Decisions (and alternatives considered)

- **Decision:** Keep e2e assertions contract-focused and lightweight.
  - **Alternative:** Deep fixture assertions against fixed row counts (rejected as brittle for evolving seed data).
- **Decision:** Load dotenv in test file.
  - **Alternative:** custom Jest setup file (deferred for now to keep diff small).

## Assumptions

- `DATABASE_URL` in `apps/api/.env` points to migrated Neon DB.

## Failure Modes

- If `.env` is missing/incorrect, `/symbols` e2e fails with table-not-found or auth errors.
- If schema changes, shape assertions may need updating.

## How to Test (commands + steps)

```bash
cd apps/api
npm run test:e2e
```

Expected:
- 3 passing tests (`/`, `/health`, `/symbols`)
