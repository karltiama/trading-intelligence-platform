## Context

The dashboard watchlist still used local dummy rows. We need a typed fetch layer for backend market summary data.

## Goal

Create a reusable helper to fetch `GET /dashboard/market-summary` using `NEXT_PUBLIC_API_URL`.

## Files Changed

- `apps/web/src/lib/api.ts`

## Implementation Notes (key logic explained)

- Added `MarketSummaryItem` type matching backend summary shape.
- Added `getMarketSummary()` with `cache: "no-store"` so dashboard reflects fresh DB data.
- Added base URL resolution with fallback `http://localhost:3001`.
- Added basic payload guard (`Array.isArray`) and clear fetch errors.

## Key Decisions (and alternatives considered)

- **Decision:** place helper in `src/lib/api.ts`.
  - **Why:** simple shared location and minimal structure changes.
  - **Alternative:** feature-local API folder (deferred to keep scope small).

## Assumptions

- Backend endpoint returns JSON array objects with watchlist-compatible fields.
- `NEXT_PUBLIC_API_URL` is available in frontend runtime environment.

## Failure Modes

- Backend unavailable returns thrown error and UI should display error state.
- Unexpected payload shape throws and should be surfaced in UI.

## How to Test (commands + steps)

1. Set `NEXT_PUBLIC_API_URL=http://localhost:3001` in web env.
2. Start backend API and run market sync.
3. Start web app and verify dashboard watchlist fetches.
