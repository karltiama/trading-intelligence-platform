## Context

Prisma commands worked, but they were ad-hoc shell commands instead of discoverable package scripts.

## Goal

Expose Prisma lifecycle commands in `apps/api/package.json` for repeatable local and CI use.

## Files Changed

- `apps/api/package.json`
- `notes/learning-log/2026-04-24/add-api-prisma-scripts.mdx`

## Implementation Notes (key logic explained)

Added scripts:
- `prisma:validate`
- `prisma:generate`
- `prisma:migrate:init`
- `prisma:studio`

## Key Decisions (and alternatives considered)

- **Decision:** Keep migration script scoped to init naming for now.
  - **Alternative:** generic migrate script with custom name flag each run (deferred for simplicity).

## Assumptions

- Developers run these scripts from `apps/api` or via root workspace forwarding scripts.

## Failure Modes

- Running `prisma:migrate:init` repeatedly after schema evolution may prompt new migrations with same intended semantic name.

## How to Test (commands + steps)

```bash
cd apps/api
npm run prisma:validate
npm run prisma:generate
```

## Next Steps

1. Add future migration scripts as needed (`prisma:migrate:dev`, `prisma:migrate:deploy`) when CI/deploy flow is added.
