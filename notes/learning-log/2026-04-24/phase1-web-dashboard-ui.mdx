## Context

Phase 0 delivered a NestJS API with `/health`, `/symbols`, `/market-data/status`, `/signals`
and a Next.js 16 + Tailwind v4 frontend scaffold. Phase 1 needs a clean dashboard
surface that we can later wire to real market data and signal endpoints. The
frontend was empty (default Next.js home page). Backend engineering is the
priority, so the frontend needs to be simple, maintainable, and built from
well-known primitives.

## Goal

Ship a responsive, dark-mode-friendly dashboard at `/dashboard` — sidebar, header,
KPIs, chart placeholder, watchlist, insights, and today's setups — using
`shadcn/ui` components and dummy data only. No backend wiring, no auth, no
chart library yet.

## Implementation Notes

- **Data-first components.** All visual components read from typed arrays in `dashboard-data.ts`. This keeps the components pure view layers and makes it trivial to replace arrays with a fetcher later.
- **`NavItem.icon`** stores the Lucide component itself (not a string key). The sidebar does `<item.icon />` directly — no icon registry, no switch statements.
- **shadcn v4 uses `@base-ui/react`** instead of Radix. Polymorphism is via `render={<Link …/>}` rather than `asChild`. The sidebar's `SidebarMenuButton` uses this pattern to render each item as a `next/link` while keeping focus/active states.
- **Active link detection** uses `usePathname()` and matches exact equality for simplicity. Fine for a flat nav; switch to `startsWith` if nested routes arrive.
- **KPI / Insight / Setup tone helpers** take a tone enum and return `{ text, bg, Icon }` objects. Keeps green/red/amber logic centralized and type-safe.
- **Chart card is a client component** only because it owns the active-timeframe `useState`. The rest of the page is a server component.
- **Watchlist color coding** uses Tailwind `emerald`/`rose` utilities instead of the shadcn `destructive` token. `destructive` is red, but red-only semantics don't cover "positive = green" — so we reach for two-sided palette colors. They read fine on both light and dark backgrounds.
- **Dark mode.** shadcn v4 ships class-based dark mode (`.dark`). We left that intact for future toggle support and appended an `@media (prefers-color-scheme: dark)` block that mirrors the `.dark` tokens onto `:root` when the OS prefers dark. No JS, no `next-themes`, no flash of wrong theme.
- **`use-mobile` hook** was regenerated by shadcn with a `setState`-in-effect pattern that fails React 19's new lint rule. Rewrote using `useSyncExternalStore`, which is the canonical way to read from `matchMedia`.

## Key Decisions (and alternatives considered)

1. **Dashboard at `/dashboard` with `/` redirecting.**
   - *Alt:* put it at `/` directly. Rejected because a marketing / landing page may live at `/` later and redirects are cheap (Next emits a 307).
2. **shadcn `Sidebar` primitive over a hand-rolled sidebar.**
   - *Alt:* custom `<aside>` with plain Tailwind. Rejected to match the project rule "prioritize shadcn/ui over custom div-heavy markup" and to get the collapsible/mobile-sheet behavior for free.
3. **Icons as component references (`LucideIcon`) in data.**
   - *Alt:* store string keys and look them up in a map. Rejected — extra indirection with no benefit at this scale.
4. **No `next-themes` yet.**
   - *Alt:* install it and add a toggle. Rejected — theme toggle wasn't in scope and Phase 1 is "dark-mode friendly", not "user-togglable theme". `prefers-color-scheme` covers the requirement with zero JS.
5. **Dummy data co-located in one file (`dashboard-data.ts`).**
   - *Alt:* one file per entity. Rejected at MVP size; promotable to a folder when real fetchers replace each array.

## Assumptions

- We're OK taking the shadcn v4 stack (Base UI + `tw-animate-css` + `lucide-react@1.11`). If a future component needs Radix-specific primitives, we can install them ad hoc.
- Timeframe selection on the chart is local state only — no URL sync, no persistence.
- All tabular numbers use `tabular-nums` so columns don't dance as values update.
- The sidebar's cookie (`sidebar_state`) persisting collapsed/expanded state is acceptable.

## Failure Modes

- **Hydration mismatch on mobile detection.** `useIsMobile` returns `false` on the server (via `getServerSnapshot`) and the real value after hydration. The sidebar renders the desktop shell server-side; on narrow screens the mobile `Sheet` replaces it post-hydration. If this flashes, we can gate the sidebar on a mounted flag — but it's fine as-is.
- **shadcn CLI overwriting `globals.css`.** If we re-run `shadcn init`, the `@media (prefers-color-scheme: dark)` block we appended will be clobbered. Keep a mental note, or factor it into a separate `@import` later.
- **Monorepo lockfile warning.** `next build` warns about multiple lockfiles (root + `apps/web`). Not fatal, but we should prune `apps/web/package-lock.json` or set `turbopack.root` in `next.config.ts` once we decide on a single source of truth.
- **Lucide v1.x is unusual.** shadcn v4 pinned `lucide-react@^1.11.0`. If we upgrade shadcn later, double-check the icon import surface hasn't changed.

## How to Test

```bash
# From repo root
npm install                      # pulls the new shadcn/base-ui deps
npm run dev:web                  # http://localhost:3000

# Build + typecheck
cd apps/web
npx tsc --noEmit
npm run lint
npx next build
```

Manual steps:

1. Hit `http://localhost:3000/` — should 307 to `/dashboard`.
2. Confirm sidebar shows `TradeIQ` + 6 items + user block.
3. Hover items when sidebar collapsed (Cmd/Ctrl+B) — tooltips should appear.
4. Click a timeframe button — selected chip changes, others go ghost.
5. Resize below `768px` — sidebar collapses into the mobile sheet (trigger in header).
6. Toggle OS dark mode — backgrounds / cards / borders flip without a reload.
7. Watchlist: AAPL/NVDA/MSFT green, SPY muted, META red.
